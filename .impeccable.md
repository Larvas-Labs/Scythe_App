## Design Context

### Users
macOS power users — primarily developers, designers, and technically literate users who want full control over what gets deleted. They run this intentionally, not by accident. They're in "maintenance mode": focused, efficient, zero tolerance for fluff.

### Brand Personality
**Precise. Surgical. Unhurried.**
A professional tool that respects the user's intelligence. Not aggressive like a cleaner app, not playful like a consumer app — closer to a refined Unix utility wrapped in a macOS native shell. Think scalpel, not broom.

### Aesthetic Direction
- **Reference**: Claude's own desktop app — clean, restrained dark UI, excellent typography hierarchy, subtle borders, purposeful whitespace.
- **Theme**: Dark default (disk utility context, developer audience, evening/night use), with a clean light mode available via toggle.
- **Anti-references**: Anything that looks like CleanMyMac (loud, marketing-heavy), anything with glow borders, anything with gradient text.
- **Signature element**: The green→blue gradient is used *only* on the StorageRing fill and action buttons — never on text.

### Design Principles
1. **Data is the hero** — sizes in JetBrains Mono, large and prominent. Everything else steps back.
2. **Earn every pixel** — no decoration that doesn't carry meaning.
3. **Consistent depth** — two surface levels (bg and surface), one border weight per context.
4. **Motion serves clarity** — only the ring animates; everything else is instant or nearly so.
5. **Neutral intelligence** — the UI feels calm even when it's about to permanently delete gigabytes.
